## How It Works

```
Morning cron job triggers
        │
        ▼
Fetch 100+ RSS feeds (parallel, <15s)
        │
        ▼
AI reads all articles at once (1M context window)
        │
        ▼
Outputs complete Markdown report + JSON metadata
        │
        ▼
Render HTML → Send email → Send Telegram → Archive
```

The AI sees everything in one pass — all articles, yesterday's events, market data, 90-day trends. No fragmented pipeline. No intermediate parsing failures.

## Architecture

```
src/
├── summarizer_v3.py    # Single-pass AI briefing generation
├── fetcher.py          # 100+ RSS concurrent fetcher
├── generator.py        # HTML report renderer
├── market_data.py      # Real-time market data (yfinance + Sina)
├── risk_scorer.py      # Quantitative risk scoring + transmission chains
├── event_graph.py      # Event extraction + deduplication
├── evolution_tracker.py # Event lifecycle + phase transition detection
├── mailer.py           # SMTP email delivery
├── context_builder.py  # Modular context assembly for AI input
├── prompt_builder.py   # Layered prompt architecture (6 layers)
├── telemetry.py        # Production metrics auto-recording
└── premium/            # Commercial features (Pro license required)
```
